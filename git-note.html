• 初始化git仓库：git init
  • 把文件修改提交到暂存区：git add [file-name]|[-A]|[--all]
  • 把暂存区内容提交到当前分支：git commit [-m] ["msg"]
  • 查看仓库状态：git status
  • 查看跟踪文件的改变情况（相比于上次commit）git diff [file-name]
  • 查看历史提交：git log
  • 退回到历史版本：git reset --hard [版本号|HEAD^(上一个)|HEAD^^(上上个)|HEAD~100(上100个)]
    • 当版本回退后，不能再通过git log查询到“未来的版本”了
    • 但你可以通过git reset --hard [版本号]"回到未来"版本。
    • 使用git reflog可以查询到你的“未来”版本号
  • 如果你在add命令后又对文件进行了修改，那么commit的内容只是你add命令之前的内容
  • 使用git checkout -- [filename]可以使文件回到上一次add或commit状态
  • 使用git reset HEAD [file-name]可以把暂存区的修改撤销，重新放回工作区
  • 所以如果你把不想被看到的修改从暂存区抹杀掉，可以git reset HEAD [file-name]再git checkout -- [filename]即可
  • 当你删除了一个文件，你可以使用git rm [file-name]把它从版本库彻底删除：如果你只是误删，可以使用git checkout -- [filename]恢复。
  • 增加远程仓库：git remote add [URL]
  • 推送到远程仓库：git push [remote-name] [local-branch]:[remote-branch]
  • 将远程库克隆到本地新目录：git clone [url]
  • 分支管理：
    • 创建分支：git branch [branch-name]
    • 删除分支：git branch -d [branch-name]
    • 切换分支：git checkout [branch-name]
    • 创建并切换分支：git checkout -b [branch-name]
    • 查看当前分支：git branch
    • 合并到当前分支：git merge [branch-name]
  • 解决冲突：合并出冲突就手动解决吧，没什么好说的
  • 把分支a合并到分支b上后，分支a的指针不变，b的指针指向二者融合的新指针的位置
